# v0.1.5

## Release Positioning

`v0.1.5` is the first release after the initial public `v0.1.0` drop that tightens the **Codex local-plugin install model** and stabilizes the **WeChat / Weixin operator workflow**.

This release stays:

- **Windows-first**
- **Codex Desktop plugin-first**
- **local runtime / local cache**
- **optional tray companion**, not tray-required

## What Changed

### Plugin install and runtime sync
- fixed local plugin install sync so the installed runtime and the versioned local-personal cache stay aligned
- fixed the versioned plugin cache bundle so plugin-contributed skills can actually surface in Codex Desktop
- removed UTF-8 BOM from skill frontmatter files that could destabilize skill parsing
- verified that plugin skills load again from the installed local-personal plugin path

### Skill surface
- restored plugin skill visibility in Codex Desktop
- skill display names now surface as:
  - `WeChat Bridge:OPS`
  - `WeChat Bridge:Task Finished Notifier`
- kept raw skill names stable and directory-aligned to match the expected plugin skill structure

### WeChat-side controls
- added `/new-session [name]`
- kept `/newsession` as a compatibility alias
- added `/restart` to restart the current bridge daemon from WeChat
- added `/test-session bind <session-id>`
- added `/test-session`
- added `/test-session unbind`
- added `/test-session quit`
- added `/record-session`
- added `/record-session add <record-name> <session-id>`
- added `/record-session delete <record-name>`
- added `/record-session clear`
- added `/use-record <record-name>`
- added `/yes` and `/no` for time-bounded confirmation flows
- added `/fallback continue` so an idle timed-out `app_server` task only switches to `exec` fallback after explicit user approval

### Session inspection and routing
- `/session` and `/sessions` now surface live session metadata more accurately
- `exec` backend session-name reporting no longer pretends to know unavailable live names
- `/use-session` success output now includes clearer session feedback
- quote-aware inbound prompts were tightened so replies are more likely to stay anchored to the quoted WeChat message

### Reply behavior
- improved segmented-output handling
- reduced broken chunk boundaries for ordered-list style progress output
- kept final-summary behavior separate from progressive output
- transient Weixin transport-side `fetch failed` sends are retried without changing the original `client_id`
- long-running `app_server` turns no longer hard-timeout into automatic fallback after 60 seconds
- the bridge now sends an idle-timeout control notice after 5 minutes of silence, keeps waiting by default, and tells the user to choose `/stop` or `/fallback continue`

### Notification workflow
- task-finished notification skill can now resolve the current Codex session more reliably
- notification template behavior was aligned with the agreed WeChat-facing format
- UTF-8-sensitive notification content now uses a payload-file flow instead of passing Chinese text directly through Windows shell arguments

## What This Release Still Supports

- private-chat receive and reply loop
- app-server primary backend with `exec` fallback
- per-contact conversation isolation
- session switching and session-name inspection
- single-message quote awareness
- inbound image and file attachment ingress into local cache
- segmented replies to avoid long silent waits
- optional final summary output
- optional Windows tray launcher and desktop shortcut
- explicit idle-timeout prompting before any `app_server` to `exec` fallback handoff

## Known Limits In v0.1.5

- private chats only
- no outbound image/file parity yet
- no pure voice-input handling yet
- no merged chat-history forwarding support
- WeChat reply context tokens are still short-lived for manual push flows
- MCP remains **stdio**, so it is managed by Codex Desktop rather than a standalone daemon

## Upgrade Notes

If you are upgrading an existing local install:

```powershell
npm install
npm run build
npm run install:plugin
```

Then fully restart Codex Desktop so the repaired local-personal plugin cache bundle is picked up.

Optional on Windows:

```powershell
npm run install:tray
```

## Recommended Description

The shortest accurate description of `v0.1.5` is:

> A Windows-first local Codex Desktop WeChat / Weixin bridge plugin with repaired local-plugin skill loading, richer session controls, inbound attachment support, and optional tray operations.

# v0.1.0

## Release Positioning

This is the first public release of **Codex WeChat Bridge**:

- a **Codex Desktop plugin**
- for **WeChat / Weixin private-chat bridging**
- with an **optional Windows tray companion**
- and a **desktop shortcut install flow**

This release is **Windows-first**. The daemon and MCP runtime are Node.js-based, but the packaged install path, tray workflow, and operator documentation are currently designed for Windows 11.

## What This Release Delivers

### Core bridge
- QR login with persisted login state
- private-chat receive and reply loop
- Codex app-server primary backend with `codex exec` fallback
- per-contact conversation isolation and session rebinding
- local diagnostics, pending-message handling, and retry-aware state
- single-message quote awareness for inbound WeChat replies
- inbound image and file attachment ingress into local cache

### WeChat-side controls
- `/help`
- `/pwd`
- `/session`
- `/newsession`
- `/use-session`
- `/append`
- `/stop`
- `/pending`
- `/model`
- `/effort`
- `/final`
- `/quota`
- `/skills`
- `/status`
- `/diagnostics`
- `/threads`
- `/sessions`
- `/ls`

### Attachments
- inbound **image** download to local cache
- inbound **file** download to local cache
- attachment metadata and local paths injected into the Codex prompt

### Windows operator experience
- optional tray launcher
- optional desktop shortcut
- daemon lifecycle controls from tray
- MCP connection reset / cleanup actions aligned with stdio MCP behavior

## Install

### For humans

```powershell
git clone <repo-url> codex-wechat-plugin
cd codex-wechat-plugin
npm install
npm run build
npm run install:plugin
```

Then:

1. Restart Codex Desktop.
2. Run `npm run login` once if the account is not already logged in.
3. Scan the QR code and send a private message from WeChat.

Optional on Windows:

```powershell
npm run install:tray
```

Then launch `WeChat Bridge` from the desktop shortcut if a tray entrypoint is desired.

### For Codex agent

```powershell
npm install
npm run build
npm run install:plugin
```

After install:

1. Restart Codex Desktop.
2. Use the local `WeChat Bridge` plugin.
3. Use the installed runtime under `~/.codex/plugins/codex-wechat-bridge`.

Optional on Windows:

```powershell
npm run install:tray
```

## Install Model

This repository is the **source of truth**.

The supported install model is:

- plugin runtime staged into:
  - `~/.codex/plugins/codex-wechat-bridge`
- marketplace entry written to:
  - `~/.agents/plugins/marketplace.json`
- Windows tray launcher installed against the **installed plugin runtime**, not the mutable development worktree

## Known Limits In v0.1.0

- private chats only
- no outbound image/file parity yet
- no inbound voice-only message support yet
- no merged chat-history forwarding support
- WeChat reply context tokens are short-lived; some manual sends and lifecycle pushes require a fresh inbound message
- MCP is **stdio**, so it is managed by Codex Desktop rather than by a standalone background daemon
- Node 22 `node:sqlite` is still experimental

- Codex WeChat plugin
- Codex Weixin plugin

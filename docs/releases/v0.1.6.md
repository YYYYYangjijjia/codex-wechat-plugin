# v0.1.6

## Release Positioning

`v0.1.6` is the stabilization release that turns outbound WeChat / Weixin delivery from a partial experiment into a usable bridge capability.

This release keeps the same project stance:

- Windows-first
- local Codex Desktop plugin-first
- no OpenClaw runtime dependency
- stdio MCP managed by Codex Desktop
- optional tray companion

## What Changed

### Outbound file and image delivery
- added real outbound local-media delivery for WeChat / Weixin private chats
- added image and general-file send support through the bridge MCP surface
- added a bundled `deliver-file` skill for Codex-side file delivery
- supported common file classes including images, txt, pdf, doc/docx-style artifacts, and zip-style attachments through the same bridge route

### Delivery authorization model
- default behavior still does **not** send files automatically
- the bridge now detects explicit user delivery intent from inbound WeChat / Weixin messages
- when delivery is explicitly requested, the bridge injects delivery guidance so Codex can send the exact artifact instead of the bridge guessing blindly
- delivery markers are now recognized so bridge fallback delivery does not duplicate files already sent by Codex

### Reply-context expiry handling
- added a persistent outbound delivery queue in bridge state
- `sendmessage ret=-2` is no longer treated as a permanent terminal failure for text and file delivery
- manual sends, notifier sends, and bridge-driven sends now enter `waiting_for_fresh_context` and replay automatically on the next fresh inbound message for that chat
- control commands also count as fresh inbound triggers, so a command like `/help` can release queued outbound deliveries once WeChat provides a new context token

### Notifier and script hardening
- `task-finished-notifier` now goes through the same bridge send path instead of bypassing bridge state
- notifier payload handling continues to enforce UTF-8 payload-file usage for Chinese and emoji content
- `deliver-file` now explicitly forbids claiming delivery after file creation/export/patch failures unless the artifact is visibly present on disk

### Weixin transport and bridge internals
- added durable outbound-delivery persistence in SQLite state
- reply orchestrator now queues final summaries and media sends when reply context expires mid-send
- bridge tool results now surface `queued` status when delivery is deferred instead of falsely reporting immediate send success
- cleaned up user-facing recovery text where bridge-generated status messages previously contained mojibake

## What This Release Supports

- WeChat / Weixin private-chat bridging into local Codex workflows
- per-contact session mapping
- explicit session switching and saved session records
- inbound image and file ingress
- segmented progress output and optional final summary
- outbound file/image delivery when explicitly requested
- deferred outbound replay after reply-context expiry
- task-finished notification delivery through the unified bridge path

## Known Limits In v0.1.6

- private chats only
- no outbound voice/file parity beyond normal file attachments
- no group-chat workflow
- no pure voice-input understanding
- no chat-history forwarding support
- reply-context refresh still depends on a fresh inbound WeChat / Weixin message; this release improves recovery semantics, not the underlying server-side context model

## Upgrade Notes

```powershell
npm install
npm run build
npm run install:plugin
```

Then fully restart Codex Desktop when validating plugin-surface or skill-surface changes.

Optional on Windows:

```powershell
npm run install:tray
```

## Recommended Description

The shortest accurate description of `v0.1.6` is:

> A Windows-first local Codex Desktop WeChat / Weixin bridge plugin with explicit outbound file delivery, queued replay for expired reply context, and unified notifier/file-send behavior.
